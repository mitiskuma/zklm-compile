How it works

The problem: An API claims to run GPT-2 on your input. How do you know it actually did? Maybe it ran a cheaper model, cached a stale answer, or made something up.

The solution: The server generates a zero-knowledge proof alongside the output. This proof is a compact cryptographic artifact (tens of KB) that mathematically guarantees the output came from running the real GPT-2 weights on your exact input. No trust required.

How verification works: Anyone can check the proof in ~4ms without re-running the model. The verifier needs only the proof, the input commitment, and the weight commitments (SHA-256 hashes of the quantized weight matrices). If the server changed even one weight or skipped one multiplication, the proof would fail.

Technical details: All arithmetic is over the Mersenne-31 field (p = 2^31 - 1). Matrix multiplications are verified via structured sumcheck protocol (Thaler 2013). LayerNorm uses a squared identity proof with QR perturbation for non-quadratic-residue cases. GELU is proved via LogUp lookup argument over a requantized INT16 table. Weights are INT8-quantized with per-matrix symmetric scaling.
